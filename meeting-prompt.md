# 미팅 노트 분석 AI 드림팀 워크플로우

> 3명의 전문 AI 에이전트가 **병렬 협업**하여 대충 적은 메모 → 깔끔한 회의록 + 액션아이템 + 후속 전략을 자동 생성합니다
> **회의록** (구조화된 정리) + **후속 제안** (다음 할 일 + 아이디어)을 한 번에 출력합니다

---

## 0. 실행 방법

### 한 줄 실행

```
meeting-prompt.md 실행해줘
```

### 필요 입력: 미팅 메모 텍스트

```
- 메모장에 대충 적은 미팅 노트를 그대로 붙여넣기
- 줄임말, 오타, 비문 모두 OK — parser가 알아서 정리합니다
- 여러 안건이 섞여 있어도 OK — 자동으로 분리합니다
```

### 입력 예시

```
3/16 고객미팅 - 네놈마켓
참석: 김대리, 박과장, 고객측 이사님

- 런칭일정 4월초로 잡자고 함
- 근데 결제모듈 아직 안됨 PG연동 이슈
- 디자인은 OK 사인받음
- 이사님이 B2B도 같이 하고싶다고... 범위 늘어날듯
- 다음주 수요일 다시 미팅
- 모바일 반응형 꼭 해달라
```

## 0.5. 사전 브리핑 (에이전트 작업 전 필수)

> 메모 텍스트를 받으면 내용을 분석하여 미팅 유형을 자동 추천합니다.
> 최소한의 질문만 하고 빠르게 에이전트를 투입합니다.

### 질문 항목 (AskUserQuestion 도구 사용)

#### Q1. 미팅 유형 확인 (자동 추천)

메모 내용을 분석하여 가장 적합한 유형을 추천합니다.

| 유형 | 설명 | 후속 제안 방향 |
|------|------|---------------|
| 고객 미팅 | 외부 고객과의 미팅 | 제안서/견적 방향, 고객 관리 전략 |
| 내부 회의 | 팀/부서 내부 논의 | 프로세스 개선, 업무 분배 최적화 |
| 킥오프 | 프로젝트 시작 미팅 | 마일스톤, 리스크 사전 식별 |
| 스프린트/데일리 | 개발 진행상황 공유 | 병목 해소, 기술 부채 관리 |
| 브레인스토밍 | 아이디어 발산 | 아이디어 우선순위화, 실현 가능성 분석 |
| 의사결정 | 특정 안건 결정 | 결정 후 실행 로드맵 |
| 회고/리뷰 | 프로젝트 회고 | 개선 액션, 반복 방지 체크리스트 |

> 팀 리드가 메모를 분석한 뒤 가장 적합한 유형을 (추천)으로 표시합니다.
> 사용자는 직접 입력도 가능합니다.

#### Q2. 집중 포인트 (선택)

```
이 미팅에서 특별히 중요하거나 집중해서 정리해줬으면 하는 부분이 있으면 알려주세요.
없으면 건너뛰어도 됩니다.
```

### 브리핑 결과 → 에이전트에 전달

사용자 답변을 정리하여 **모든 에이전트 프롬프트에 아래 블록을 추가**합니다:

```
## 사전 브리핑 (사용자 지정)
- 미팅 유형: [확인된 유형]
- 집중 포인트: [있으면 내용, 없으면 "없음"]

⚠️ 위 브리핑 내용을 반드시 반영하여 작업하세요.
```

## 1. 팀 아키텍처

### 에이전트 구성 (3명)

```
┌─────────────────────────────────────────────────┐
│              team-lead (팀 리드)                   │
│    입력 판별, 태스크 관리, 최종 통합, 노션 연동      │
├─────────────────────────────────────────────────┤
│                                                  │
│  [1] parser    → 대충 메모 → 구조화된 회의록        │
│  [2] analyst   → 결정/액션/이슈 추출      ← 병렬   │
│  [3] advisor   → 후속 전략 + 아이디어     ← 병렬   │
│                                                  │
└─────────────────────────────────────────────────┘
```

### 태스크 의존성 & 병렬 처리 구조

```
[Task #1] 메모 파싱 + 회의록 구조화 (parser)
    │
    ├──────────────────┐
    ▼                  ▼
[Task #2]          [Task #3]         ← #1 완료 후 2개 동시 병렬
결정/액션/이슈      후속 전략/아이디어
(analyst)          (advisor)
    │                  │
    └────────┬─────────┘
             ▼
[Task #4] 최종 통합 + 검수 + 노션 연동 (team-lead)  ← #2, #3 모두 완료 후
```

## 2. 팀 생성 & 태스크 설정 절차

### STEP 1: output 디렉토리 자동 생성

```bash
PROJECT_DIR="output/meeting/$(date +%Y%m%d)_[미팅제목]"
mkdir -p "$PROJECT_DIR/.work"
```

**폴더 구조:**
```
output/meeting/[YYYYMMDD]_[미팅제목]/
├── minutes.md              ← 통합 회의록 (사용자용)
├── minutes.html            ← 브라우저용 HTML (공유/인쇄용)
└── .work/                  ← 중간 파일 (완료 후 삭제 가능)
    ├── 01_parsed.md        ← 파싱된 회의록
    ├── 02_analysis.md      ← 결정/액션/이슈 분석
    └── 03_advice.md        ← 후속 전략/아이디어
```

### STEP 2: 태스크 4개 생성 + 의존성 설정

| Task ID | Subject | Owner | Blocked By | Output File |
|---------|---------|-------|------------|-------------|
| #1 | 메모 파싱 + 회의록 구조화 | parser | - | `{PROJECT_DIR}/.work/01_parsed.md` |
| #2 | 결정/액션/이슈 추출 | analyst | #1 | `{PROJECT_DIR}/.work/02_analysis.md` |
| #3 | 후속 전략 + 아이디어 | advisor | #1 | `{PROJECT_DIR}/.work/03_advice.md` |
| #4 | 최종 통합 + 검수 + HTML + 노션 | team-lead | #2, #3 | `{PROJECT_DIR}/minutes.md`, `minutes.html` |

### STEP 3: 에이전트 순차/병렬 투입

```
1단계: parser 투입 → Task #1 완료 대기
2단계: analyst + advisor 동시 투입 (2개 병렬) → Task #2, #3 모두 완료 대기
3단계: 팀 리드가 직접 Task #4 수행 (통합 + 검수 + HTML + 노션)
```

### STEP 4: 정리 & 완료

```
1. .work/ 폴더 삭제 (중간 파일 보존 옵션 선택 시 유지)
2. 사용자에게 안내:
   - output/meeting/[날짜]_[미팅제목]/minutes.md (통합 회의록)
   - output/meeting/[날짜]_[미팅제목]/minutes.html (브라우저에서 미리보기)
3. (노션 MCP 연동 시) 노션 데이터베이스에 자동 업로드
```

### [Agent 1] parser — 메모 파싱 + 회의록 구조화

**subagent_type:** `general-purpose`

**Task Prompt:**
```
당신은 회의록 정리 전문가입니다.
대충 적은 미팅 메모를 깔끔하고 구조화된 회의록으로 변환하는 것이 임무입니다.

## 할 일
1. 사용자가 입력한 미팅 메모 원문을 읽으세요
2. 내용을 분석하고 구조화하세요
3. 정리된 회의록을 저장하세요

## 파싱 원칙

### 텍스트 교정
- 줄임말 → 풀어쓰기 (예: "개발 ㄱㄱ" → "개발 진행하기로 함")
- 오타/비문 → 자연스러운 문장으로 교정
- 구어체 → 간결한 서술체로 변환
- 단, 원문의 의미를 절대 변경하지 않음

### 구조화
- 시간순 흐름 파악 및 재구성
- 안건별 섹션 분리 (논의 주제가 바뀌는 지점 감지)
- 발언자 구분 (메모에 표시되어 있는 경우)
- 관련 없는 내용끼리 묶이지 않도록 주의

### 불확실한 부분 처리
- 의미가 불분명한 부분: [?] 표시
- 추측이 필요한 부분: [추정: ...] 표시
- 누락된 것 같은 정보: [확인 필요: ...] 표시

### [Agent 3] advisor — 후속 전략 + 아이디어 (병렬 B)

**subagent_type:** `general-purpose`
**blocked_by:** Task #1

**Task Prompt:**
```
당신은 15년 경력의 시니어 컨설턴트이자 전략 기획자입니다.
회의 내용을 바탕으로 "다음에 뭘 해야 하는지"를 제안하는 것이 임무입니다.
단순 정리가 아니라, 회의에서 놓친 관점과 새로운 아이디어를 제시하세요.

## 할 일
1. {PROJECT_DIR}/.work/01_parsed.md 를 먼저 읽으세요
2. 회의 내용을 깊이 분석하세요
3. 후속 전략과 아이디어를 작성하세요

## 제안 항목

### 1. 다음 미팅 안건 제안
이번 미팅의 미결 이슈와 흐름을 고려하여:
- 다음 미팅에서 반드시 다뤄야 할 안건 (2-3개)
- 각 안건별 사전 준비물
- 추천 미팅 형식 (전체/소규모/1:1)

### 2. 후속 전략 제안
이번 미팅에서 논의된 방향에 대해:
- 현재 방향의 장단점 분석
- 대안 또는 보완 전략 (있으면)
- 우선순위 재조정이 필요한 부분

### 3. 놓친 관점
미팅에서 다루지 않았지만 고려해야 할 것:
- 빠진 이해관계자/의견
- 간과된 리스크
- 장기적 관점에서의 고려사항

### 4. Quick Win — 바로 실행 가능한 것
미팅 직후 바로 할 수 있는 작은 액션:
- 5분 안에 할 수 있는 것 (예: 메일 한 통, 슬랙 메시지)
- 오늘 중으로 할 수 있는 것
- 이번 주 안에 할 수 있는 것

### 5. 아이디어 / 인사이트
미팅 내용에서 영감을 받은 새로운 아이디어:
- 프로세스 개선 아이디어
- 기술적 제안 (개발 관련 미팅인 경우)
- 비즈니스 기회 (고객 미팅인 경우)

## 미팅 유형별 맞춤 제안

### 고객 미팅인 경우
- 고객 기대치 관리 전략
- 제안서/견적 방향 제안
- 후속 커뮤니케이션 타이밍

### 내부 회의인 경우
- 업무 분배 최적화 제안
- 프로세스 병목 해소 방안
- 팀 협업 개선점

### 킥오프인 경우
- 마일스톤 제안
- 리스크 사전 식별
- 커뮤니케이션 계획

### 브레인스토밍인 경우
- 아이디어 우선순위 매트릭스 (영향도 x 실현 가능성)
- 다음 단계: 프로토타입 or 추가 리서치

## 산출물 ({PROJECT_DIR}/.work/03_advice.md)
위 제안 항목을 마크다운으로 작성하세요.
실무자가 읽고 바로 행동할 수 있도록 구체적으로.
미팅 유형에 맞는 섹션을 우선 배치하세요.

### [Task #4] 팀 리드 직접 수행 — 최종 통합 + 검수 + HTML

에이전트를 추가 투입하지 않고 **팀 리드가 직접** 아래를 수행합니다:

```
## 할 일
1. .work/ 폴더의 3개 파일을 모두 읽기
2. 통합 검수 수행 (아래 체크리스트)
3. 검수 결과를 반영하여 최종 minutes.md 작성
4. minutes.html 생성

## 검수 체크리스트 (PASS/FAIL)
- [ ] 원본 메모 대비 누락된 내용 없음
- [ ] 결정사항과 액션아이템이 정확히 분리됨
- [ ] 액션아이템에 담당자/기한/우선순위 모두 표시
- [ ] 미결 이슈에 다음 단계가 명시됨

## minutes.html 생성
깔끔하고 공유하기 좋은 HTML 페이지를 생성합니다:

### 디자인 원칙
- 다크 테마 기본 (배경: #1a1a2e, 텍스트: #e0e0e0)
- 상단 메타 카드 (한눈에 회의 개요 파악)
- 액션아이템 체크박스 (클릭하면 체크/해제 — localStorage 저장)
- 우선순위 컬러 코딩 (🔴🟡🟢)
- 인쇄 최적화 (@media print — 흰 배경, 체크박스 → 빈 칸)
- 반응형 (모바일에서도 보기 편하게)
- 가로 최대 800px, 가운데 정렬
- 순수 HTML+CSS+JS (외부 의존 없음)
- 브라우저에서 더블클릭으로 바로 열기 가능

### HTML 구조
```html
<!DOCTYPE html>
<html lang="ko">
<head>
    <meta charset="UTF-8">
    <title>[미팅 제목] — 회의록</title>
    <style>
        /* 다크 테마 CSS */
        /* 메타 카드 */
        /* 안건 섹션 */
        /* 결정사항 하이라이트 */
        /* 액션아이템 테이블 + 체크박스 */
        /* 우선순위 컬러 (🔴🟡🟢) */
        /* 미결이슈 경고 스타일 */
        /* 후속 제안 카드 */
        /* Quick Win 체크리스트 */
        /* @media print (흰 배경, 깔끔한 인쇄) */
        /* 반응형 */
    </style>
</head>
<body>
    <header>
        <!-- 미팅 제목 -->
        <!-- 메타 정보 카드 (일시/참석자/유형) -->
    </header>
    <main>
        <section id="discussion">
            <!-- 안건별 논의 -->
        </section>
        <section id="decisions">
            <!-- 결정 사항 (하이라이트 박스) -->
        </section>
        <section id="actions">
            <!-- 액션 아이템 테이블 (체크박스 포함) -->
        </section>
        <section id="issues">
            <!-- 미결 이슈 (경고 스타일) -->
        </section>
        <section id="numbers">
            <!-- 주요 숫자/일정 -->
        </section>
        <section id="followup">
            <!-- 후속 제안 -->
            <!-- Quick Win 체크리스트 -->
            <!-- 아이디어 -->
        </section>
    </main>
    <footer>
        <button onclick="window.print()">🖨️ 인쇄</button>
    </footer>
    <script>
        // 액션아이템 체크박스 토글 + localStorage 저장
        // Quick Win 체크박스
        // 인쇄 버튼
    </script>
</body>
</html>
```

## 4. 노션 연동 (MCP 설정 후 활성화)

> 노션 MCP가 설정되어 있으면 자동으로 노션에 업로드합니다.
> MCP가 없으면 이 단계는 건너뜁니다.

### 노션 업로드 절차

```
1. 노션 MCP 도구로 미팅 데이터베이스 확인
2. 새 페이지 생성 (데이터베이스에 추가)
3. 프로퍼티 매핑:
   - 제목: [미팅 제목]
   - 미팅 유형: [유형]
   - 참석자: [이름들]
   - 일시: [날짜]
   - 상태: 정리완료
4. 페이지 본문에 minutes.md 내용 삽입
5. 사용자에게 노션 페이지 URL 안내
```

### 노션 MCP 설정 방법 (최초 1회)

```
1. https://www.notion.so/profile/integrations 에서 인테그레이션 생성
2. 시크릿 토큰 (ntn_...) 복사
3. Claude Code에서 MCP 서버 추가:
   claude mcp add notion-mcp -- npx -y @notionhq/notion-mcp-server
4. 환경변수에 토큰 설정
5. 노션에서 미팅 데이터베이스 페이지에 인테그레이션 연결
```

## 5. 최종 산출물

### 사용자에게 전달되는 최종 파일 (2개)

```
output/meeting/[YYYYMMDD]_[미팅제목]/
├── minutes.md     ← 통합 회의록 (회의록+액션+후속제안 all-in-one)
└── minutes.html   ← 브라우저용 HTML (공유/인쇄용)
```

> **minutes.md** = 노션에 바로 붙여넣기, 전체 내용 한눈에 확인
> **minutes.html** = 브라우저에서 예쁘게 보기, 체크박스로 액션 관리, 인쇄 가능

### 실행 흐름 요약

```
STEP 1: Claude Code에 "meeting-prompt.md 실행해줘" + 미팅 메모 붙여넣기
        → 미팅 유형 자동 판별 → 3명 에이전트 협업 → 최종 산출물 자동 생성

STEP 2: minutes.md로 회의록 + 액션아이템 확인
        → minutes.html을 브라우저에서 열어 체크박스로 진행 관리
        → 필요하면 인쇄하여 공유

STEP 3: (노션 MCP 설정 시) 자동으로 노션에 업로드
        → 미팅 유형별로 데이터베이스 자동 정리
```

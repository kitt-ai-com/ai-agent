## 1. 제안 배경 및 목적

### 1.1 고객의 비즈니스 과제

국내 B2B 농수축산물 유통 시장은 약 150조원 규모에 달하지만, 온라인 전환율은 5~8%에 불과합니다. 대다수의 유통 사업자는 여전히 카카오톡, 전화, 엑셀에 의존한 수작업 프로세스를 유지하고 있으며, 이로 인해 다음과 같은 구조적 비효율이 반복되고 있습니다.

| 업무 영역 | 현재 문제점 | 비즈니스 영향 |
|-----------|------------|--------------|
| **주문 접수** | 셀러별 채널(쿠팡, 스마트스토어 등)의 엑셀 양식이 모두 다름. 수작업 취합 | 주문 처리 지연, 입력 오류 빈발 |
| **발주 관리** | 공급사마다 상이한 엑셀 양식에 맞춰 개별 발주서 작성 | 관리자 업무 시간의 40% 이상 소모 |
| **송장 처리** | 공급사 회신 송장을 주문 건별로 수동 매칭 | 오매칭 발생, 배송 지연 |
| **가격 관리** | 농수축산물 특성상 일일 가격 변동. 개별 통보 및 반영 지연 | 마진 손실, 셀러 불만 |
| **정산 관리** | 수작업 정산, 클레임(반품/환불) 반영 누락 | 정산 분쟁, 신뢰 저하 |

이러한 비효율은 단순히 시간과 비용의 문제를 넘어, 사업 확장의 병목으로 작용하고 있습니다. 셀러 수가 늘어날수록 관리자의 반복 업무가 비례 증가하며, 이를 사람의 노동으로 해결하는 방식은 명확한 한계가 있습니다.

### 1.2 핵심 가치 제안 (Value Proposition)

> **네놈마켓은 AI 채팅을 메인 인터페이스로 활용하여, B2B 농수축산물 유통의 주문-발주-배송-정산 전 과정을 자연어 명령만으로 처리할 수 있는 통합 관리 플랫폼입니다.**
>
> 관리자는 "CJ공급사 발주서 뽑아줘"라는 한마디로 발주서를 생성하고, 셀러는 어떤 양식의 엑셀이든 채팅창에 드래그 앤 드롭하는 것만으로 주문을 등록할 수 있습니다.

### 1.3 제안 목적

본 제안은 네놈마켓 B2B 플랫폼의 웹 애플리케이션 및 AI 채팅 시스템을 설계, 개발, 안정화하여 고객사에 인도하는 것을 목적으로 합니다.

- **반복 업무 자동화**: 엑셀 기반 수작업을 AI 자동화로 전환하여 관리자 업무 부하 70% 이상 절감
- **데이터 일원화**: 파편화된 주문-배송-정산 데이터를 단일 플랫폼으로 통합하여 정합성 확보
- **확장 가능한 기반**: 초기 빠른 론칭 후, 트래픽 증가에 따라 유연하게 확장할 수 있는 아키텍처 구축

## 3. 프로젝트 핵심 차별점

### 3.1 AI 채팅 기반 업무 처리 — 자연어가 곧 시스템 명령

기존 B2B 유통 시스템은 복잡한 메뉴 탐색과 폼 입력을 요구합니다. 네놈마켓은 AI 채팅을 **메인 인터페이스**로 채택하여, 사용자가 자연어로 업무를 처리할 수 있습니다.

```
관리자: "감귤류 전체 10% 인상해줘"
🤖 AI: 감귤류 12개 상품의 가격을 10% 인상합니다.
       - 제주감귤 5kg: 25,000원 → 27,500원
       - 한라봉 3kg: 35,000원 → 38,500원
       ... (10개 더)
       적용하시겠습니까?
관리자: "응"
🤖 AI: 12개 상품 가격 인상 완료. 해당 셀러들에게 알림을 보낼까요?
```

Claude API의 Tool Use(Function Calling) 기능을 활용하여 자연어 입력을 시스템 API 호출로 변환합니다. 관리자와 셀러는 역할에 따라 실행 가능한 명령이 구분되며, 권한 밖의 요청은 자동으로 차단됩니다.

**비즈니스 임팩트**: 비개발자도 시스템 교육 없이 즉시 업무 처리 가능. 기존 대시보드 방식 대비 업무 처리 속도 3~5배 향상.

### 3.2 엑셀 자동 매핑 — 어떤 양식이든 AI가 인식

B2B 유통의 현실은 표준화되지 않은 엑셀입니다. 쿠팡, 스마트스토어, 자체몰 각각의 주문 엑셀 양식이 다르고, 공급사마다 발주서/송장 양식이 다릅니다.

네놈마켓의 AI 엑셀 매핑 엔진은 이 문제를 근본적으로 해결합니다.

```
[셀러가 쿠팡 주문 엑셀 업로드]
🤖 AI: 쿠팡 주문 엑셀을 인식했습니다.
       컬럼 매핑 결과:
       - "주문번호" → 주문번호
       - "수취인명" → 수취인
       - "수취인연락처" → 연락처
       - "배송지" → 배송주소
       - "상품명" → 상품 (자동 매칭: 제주감귤 5kg)
       - "수량" → 수량
       총 15건 주문 등록하시겠습니까?
```

- **입력**: 어떤 양식의 엑셀이든 업로드
- **처리**: Claude API가 컬럼 헤더와 데이터 패턴을 분석하여 시스템 필드에 자동 매핑
- **출력**: 공급사별 맞춤 양식의 발주서 엑셀 자동 생성

**비즈니스 임팩트**: 엑셀 양식 불일치로 인한 수작업 제거. 셀러 온보딩 시간 90% 단축.

### 3.3 역할 기반 접근 제어 — 관리자/셀러 권한 분리

네놈마켓은 단순한 접근 제어를 넘어, AI 채팅 명령 수준까지 역할별 권한을 분리합니다.

| 기능 영역 | 플랫폼 관리자 | 셀러 |
|-----------|:---:|:---:|
| 상품 등록/수정/가격 변경 | O | X |
| 상품 조회/다운로드 | O | O |
| 주문 전체 조회 | O | 본인 것만 |
| 발주서 생성/다운로드 | O | X |
| 정산 조회 | 전체 | 본인만 |
| 클레임 승인/처리 | O | 요청만 |
| 공지 발송 | O | X |

셀러 등급 체계(골드/실버/브론즈 등)에 따라 상품 가격과 수수료율이 자동으로 차등 적용되며, 특정 셀러에 대한 개별 특가 설정도 가능합니다.

**비즈니스 임팩트**: 셀러가 자신의 권한 범위 내에서 자유롭게 시스템을 활용하면서도, 핵심 관리 기능은 관리자만 접근할 수 있어 데이터 보안과 운영 효율을 동시에 확보합니다.

## 1. UX 설계 원칙

네놈마켓의 UX 설계는 다음 5가지 원칙을 기반으로 합니다.

### 원칙 1: AI 채팅 우선, 대시보드 보조 (Chat-First Hybrid)

AI 채팅이 업무의 주요 진입점이며, 대시보드는 데이터 조회와 시각적 확인을 위한 보조 수단입니다. 사용자는 채팅으로 빠르게 업무를 처리하고, 필요할 때 대시보드에서 상세 내용을 확인합니다.

- 모든 화면에서 AI 채팅 패널에 즉시 접근 가능
- 채팅에서 시작한 작업을 대시보드에서 이어서 확인 가능 (딥링크 연동)

### 원칙 2: 3클릭 이내 목표 도달

관리자와 셀러 모두 주요 업무를 3클릭(또는 1문장) 이내에 시작할 수 있도록 정보 구조를 설계합니다.

- 대시보드에서 핵심 지표를 한눈에 파악
- 좌측 사이드바에서 주요 메뉴에 즉시 접근
- AI 채팅으로 클릭 없이 자연어 명령 실행

### 원칙 3: 일관된 패턴과 예측 가능한 인터랙션

동일한 유형의 기능은 동일한 UI 패턴을 사용합니다. 목록 조회, 상세 보기, 폼 입력, 엑셀 업/다운로드 등 반복되는 인터랙션은 통일된 컴포넌트로 구현하여 학습 곡선을 최소화합니다.

- shadcn/ui 기반 디자인 시스템으로 컴포넌트 일관성 확보
- 데이터 테이블, 필터, 페이지네이션 등 공통 패턴 표준화

### 원칙 4: 실시간 피드백과 상태 가시성

사용자의 모든 행동에 대해 즉각적인 피드백을 제공하고, 현재 상태를 명확히 표시합니다.

- 엑셀 업로드 진행률 표시
- 주문/배송 상태 변경 시 실시간 반영 (WebSocket)
- AI 채팅 응답 스트리밍 (토큰 단위 실시간 표시)

### 원칙 5: 오류 방지와 복구 용이성

중요한 작업(가격 변경, 일괄 처리, 삭제 등)은 실행 전 확인 단계를 두고, 실행 후에도 되돌릴 수 있는 방법을 제공합니다.

- AI 채팅의 일괄 작업은 실행 전 미리보기 + 확인 프로세스 적용
- 가격 변경, 상태 변경 등 주요 작업 이력 기록 (ActivityLog)

### 2.1 디자인 키워드

| 키워드 | 의미 | UI 반영 |
|--------|------|---------|

| **신뢰** | B2B 거래의 정확성과 안정감 | 깔끔한 타이포그래피, 데이터 정합성 표시, 정산 투명성 |

### 2.2 톤앤매너

**B2B 전문성 + 친근한 AI 인터페이스**

- **컬러 체계**: 네이비 블루 계열 (신뢰, 전문성) + 그린 계열 포인트 (농수축산물 도메인, 성장)
- **타이포그래피**: Pretendard (한글 가독성 우수, 숫자 정렬 탁월) + 모노스페이스 (금액/수량)
- **아이콘**: Lucide Icons (shadcn/ui 기본 아이콘 세트, 간결하고 일관된 라인 스타일)
- **레이아웃**: 넉넉한 여백, 카드 기반 정보 그룹핑, 데이터 밀도와 가독성의 균형
- **AI 채팅 영역**: 밝은 배경 + 부드러운 말풍선, 친근하면서도 정확한 톤의 AI 응답

## 3. 기대 효과

### 3.1 정량적 기대 효과

| 효과 항목 | 현재 (As-Is) | 도입 후 (To-Be) | 개선율 |
|-----------|-------------|-----------------|--------|
| **주문 처리 시간** | 건당 평균 15분 (수작업 취합) | 건당 평균 2~3분 (AI 자동 처리) | **70~80% 단축** |
| **발주서 생성 시간** | 공급사당 30분 (양식 맞춤 수작업) | 공급사당 1분 (AI 자동 생성) | **95% 단축** |
| **엑셀 데이터 입력** | 양식별 수동 매핑 필요 | AI 자동 인식/매핑 | **수작업 80% 감소** |
| **송장 매칭 오류율** | 수작업 매칭 시 5~10% 오류 | AI 자동 매칭 시 1% 미만 | **오류율 90% 감소** |
| **셀러 온보딩 시간** | 시스템 교육 2~3일 | AI 채팅으로 즉시 사용 | **80% 단축** |
| **정산 처리 시간** | 월 정산에 3~5일 소요 | 자동 정산 1일 이내 | **70% 단축** |
| **관리자 반복 업무** | 일 8시간 중 5시간 반복 업무 | 자동화로 1~2시간 | **60~75% 절감** |

### 3.2 정성적 기대 효과

#### 고객 만족도 향상
- 셀러가 AI 채팅으로 주문 상태, 송장 정보, 정산 내역을 실시간으로 확인할 수 있어 문의 빈도 감소
- 가격 변동, 품절 안내 등 자동 알림으로 셀러와의 커뮤니케이션 품질 향상
- 클레임 처리 투명성 확보로 셀러 신뢰도 제고

#### 데이터 기반 의사결정
- 실시간 대시보드로 매출, 판매 트렌드, 재고 현황을 한눈에 파악
- 상품별/셀러별/기간별 통계 분석을 통한 전략적 의사결정 지원
- AI 이상 탐지를 통한 리스크 사전 감지 (주문 급증, 높은 반품률 등)

#### 사업 확장성
- 셀러 수 증가에도 관리자 업무 부하가 비례 증가하지 않는 구조
- AI 채팅 기반 직관적 UX로 신규 셀러 진입 장벽 최소화
- 확장 가능한 아키텍처로 카테고리 확대, 모바일 앱, 공급사 포털 등 단계적 성장 가능

#### 경쟁 우위 확보
- 국내 B2B 농수축산물 유통 시장에서 AI 채팅 기반 플랫폼은 업계 최초
- 엑셀 자동 매핑 기능으로 기존 솔루션 대비 명확한 기술적 차별점 보유
- 선점 효과를 통한 시장 지배력 확보 가능

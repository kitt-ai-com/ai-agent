## 1. 프로젝트 개요

### 1.1 비즈니스 배경

농수축산물 B2B 유통 시장은 가격 변동이 빈번하고, 다수의 판매 채널(쿠팡, 네이버 스마트스토어 등)과 공급사를 관리해야 하는 복잡한 구조입니다. 현재 대부분의 도매업체는 **엑셀, 카카오톡, 전화**로 주문·발주·정산을 처리하며, 다음과 같은 문제를 겪고 있습니다:

| 문제점 | 현황 | 연간 손실 |
|--------|------|----------|
| **수작업 주문 처리** | 채널별 주문 엑셀 수동 취합 | 인건비 월 200만원+ |
| **발주서 작성 지연** | 공급사별 양식 맞춰 수동 작성 | 발주 지연 → 품절 리스크 |
| **송장 매칭 오류** | 공급사 송장을 수동으로 주문에 매칭 | 오배송/CS 비용 증가 |
| **정산 누락** | 채널별 수수료 수동 계산 | 정산 오류 월 평균 5% |
| **실시간 재고 파악 불가** | 재고 현황 엑셀 관리 | 과잉/부족 재고 손실 |

### 1.2 솔루션 제안: 네놈마켓

**"발주부터 정산까지, B2B 유통의 모든 과정을 하나의 플랫폼에서"**

네놈마켓은 **판매자(셀러)와 플랫폼 관리자** 간 주문·배송·정산 업무를 자동화하는 **웹 기반 B2B 유통 관리 SaaS**입니다.

#### 핵심 가치 제안

1. **주문 자동화**: 쿠팡/스마트스토어 등 판매 채널 주문을 엑셀 업로드 한 번으로 일괄 등록
2. **발주서 자동 생성**: 공급사별 맞춤 양식으로 발주서 자동 생성 (엑셀 다운로드)
3. **송장 자동 매칭**: 공급사 송장 엑셀 업로드 시 주문별 자동 매칭 → 셀러에게 즉시 반영
4. **등급별 차등 가격**: 셀러 등급/개별 협의에 따른 상품 가격 자동 적용
5. **정산 자동화**: 등급별 수수료, 반품/환불 자동 계산 → 정산 내역 즉시 확인

### 1.3 기대 효과

| 효과 | 정량적 지표 |
|------|-----------|
| 주문 처리 시간 | 80% 단축 (4시간 → 50분) |
| 발주 정확도 | 오류율 5% → 0.5% |
| 정산 자동화 | 수작업 제로, 실시간 정산 가능 |
| 재고 관리 | 실시간 재고 현황 → 품절 손실 70% 감소 |
| 인건비 절감 | 월 200만원+ (주문 처리 인력 재배치) |

## 2. 시장 분석 및 경쟁 우위

### 2.1 시장 현황

| 구분 | 내용 |
|------|------|
| 시장 규모 | 국내 B2B 유통 시장 약 420조원 (2025년 기준) |
| 성장률 | 연평균 12% 성장 (이커머스 확대) |
| 주요 고객 | 중소 도매업체, 쿠팡/스마트스토어 판매자 (약 80만 명) |
| 기술 도입률 | 수작업 비중 70% (디지털화 낮음) |

### 2.2 경쟁사 비교 분석

| 구분 | 발주모아 | 도매꾹 | **네놈마켓** |
|------|---------|--------|------------|
| **타겟** | 판매사(공급사 통합) | 소매점↔도매 | **셀러↔플랫폼** |
| **주문 처리** | 수동 입력 | 수동 입력 | **엑셀 자동 업로드** |
| **발주서 생성** | ✅ 지원 | ❌ | **✅ 공급사별 양식** |
| **송장 매칭** | 수동 | 수동 | **✅ 자동 매칭** |
| **등급별 가격** | ❌ | ✅ | **✅ + 개별 설정** |
| **정산 자동화** | ✅ 기본 | ❌ | **✅ 수수료+클레임** |
| **재고 관리** | ❌ | ✅ | **✅ 실시간** |

### 2.3 차별화 포인트

1. **엑셀 기반 대량 처리**: 판매 채널 주문을 엑셀로 그대로 업로드 → 즉시 등록 (수작업 최소화)
2. **공급사 발주 자동화**: 공급사별 맞춤 양식 발주서 자동 생성 (발주 지연 제로)
3. **송장 자동 매칭 로직**: 공급사 송장 엑셀 업로드 시 주문번호/상품명 기반 자동 매칭
4. **유연한 가격 정책**: 등급별 기본 가격 + 셀러별 개별 협의 가격 동시 지원
5. **확장 가능성**: Phase 4+ AI 채팅으로 자연어 업무 처리 (향후 차별화)

### 3.2 시스템 아키텍처

```
┌─────────────────────────────────────────────────────────────┐
│                        Client Layer                          │
│   ┌──────────────┐           ┌──────────────┐              │
│   │ 관리자 대시보드 │           │ 셀러 대시보드  │              │
│   │ (Next.js SPA) │           │ (Next.js SPA) │              │
│   └──────────────┘           └──────────────┘              │
└──────────────────────┬──────────────────────────────────────┘
                       │ HTTPS / REST API
┌──────────────────────▼──────────────────────────────────────┐
│                    Application Layer                         │
│  Next.js 15 App Router + API Routes (TypeScript)            │
│  ┌────────────┬────────────┬────────────┬────────────┐     │
│  │ 상품 서비스  │ 주문 서비스  │ 정산 서비스  │ 인증 서비스  │     │
│  └────────────┴────────────┴────────────┴────────────┘     │
│  ┌──────────────────────────────────────────────────┐     │
│  │  엑셀 처리 (xlsx) | 송장 자동 매칭 | 발주서 생성   │     │
│  └──────────────────────────────────────────────────┘     │
└──────────────────────┬──────────────────────────────────────┘
                       │ Prisma ORM
┌──────────────────────▼──────────────────────────────────────┐
│                      Data Layer                              │
│  PostgreSQL (Supabase) - 상품/주문/정산/회원/공급사 테이블    │
│  Supabase Storage - 사업자등록증, 엑셀 파일 저장             │
└─────────────────────────────────────────────────────────────┘
```

### 3.3 기술 스택 상세

| 구분 | 기술 | 선정 사유 |
|------|------|----------|
| **프론트엔드** | Next.js 15 (App Router) | SSR/CSR 하이브리드, SEO 최적화, 빠른 초기 로딩 |
| **언어** | TypeScript | 타입 안정성, 대규모 코드베이스 유지보수성 |
| **UI/UX** | Tailwind CSS + shadcn/ui | 일관된 디자인 시스템, 반응형 UI 빠른 구현 |
| **백엔드** | Next.js API Routes | 프론트와 동일 코드베이스 → 개발 속도 향상 |
| **데이터베이스** | PostgreSQL (Supabase) | 관계형 DB (주문-상품-셀러 복잡한 관계), ACID 보장 |
| **ORM** | Prisma | 타입 안전 쿼리, 마이그레이션 자동화 |
| **인증** | NextAuth.js | OAuth + 이메일 로그인, 역할 기반 권한 관리 |
| **파일 처리** | xlsx 라이브러리 | 엑셀 읽기/쓰기, 대량 데이터 처리 |
| **배포** | Vercel (웹) + Supabase (DB) | 자동 CI/CD, 제로 다운타임, 글로벌 CDN |

> **Phase 4+ 확장 가능**: AWS ECS/RDS/S3로 마이그레이션 용이 (동일한 코드베이스)

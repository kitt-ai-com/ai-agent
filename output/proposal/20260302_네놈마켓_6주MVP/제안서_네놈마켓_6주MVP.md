# 네놈마켓 B2B 유통 플랫폼 — 6주 MVP 개발 제안서

**제안일자:** 2026년 3월 2일
**제안 범위:** Phase 1 MVP (핵심 미구현 기능 완성)
**개발 기간:** 6주
**제안 금액:** 5,000만원 ~ 1억원 (VAT 별도)

---

## 1. 프로젝트 개요

### 1.1 배경 및 목적

네놈마켓 B2B 유통 플랫폼은 현재 **68개 기능 중 29개(42.6%)가 구현 완료** 상태입니다. 상품 관리, 주문 기본 CRUD, 회원 관리 등 기반 기능은 구축되어 있으나, **정산/매출 관리**, **CS 관리**, **통계 대시보드**, **송장 자동 매칭** 등 비즈니스 핵심 기능이 미구현 상태입니다.

본 제안은 **6주 집중 개발**을 통해 플랫폼의 **실제 운영이 가능한 MVP(Minimum Viable Product)** 를 완성하는 것을 목표로 합니다.

### 1.3 6주 MVP 전략

> **"운영 가능한 플랫폼"** 을 최우선 목표로, 비즈니스 임팩트가 큰 기능을 선별하여 집중 개발합니다.

**핵심 원칙:**
1. **수익 직결 기능 우선** — 정산/매출 관리 없이는 비즈니스 운영 불가
2. **운영 효율 기능 선택** — 송장 자동 매칭, CS 관리로 일상 운영 지원
3. **AI 채팅은 Phase 2로 이관** — 6주 내 AI 채팅 시스템 구축은 품질 리스크가 높음
4. **외부 API 연동 최소화** — 택배사 API, 카카오 알림톡은 Phase 2로 이관

### 3.2 신규 구현 핵심 로직

#### 정산 계산 엔진
```
주문 완료 건 조회 (기간별)
  → 셀러별 매출 집계 (SUM orderItem.totalPrice)
  → 수수료 적용 (등급 수수료 OR 개별 수수료, 우선순위: 개별 > 등급)
  → 클레임 차감 (반품/환불 금액)
  → 순 정산액 = 매출 - 수수료 - 클레임 차감
  → Settlement 레코드 생성
```

#### 송장 자동 매칭 로직
```
공급사 송장 엑셀 업로드
  → 엑셀 파싱 (주문번호/상품명/송장번호 컬럼 인식)
  → OrderItem 테이블과 매칭 (주문번호 + 상품 기준)
  → 매칭 성공: trackingNo 업데이트, deliveryStatus → SHIPPING
  → 매칭 실패: 미매칭 목록 반환 (관리자 수동 처리)
  → Invoice 레코드에 매칭 결과 기록
```

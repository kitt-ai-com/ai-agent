# 요구분석 및 기술설계 문서

## 사전 브리핑
- **프로젝트명**: 불용자재 AI 매칭 플랫폼
- **프로젝트 유형**: 반응형 웹 + 관리자
- **기술 스택**: Next.js + Supabase + OpenAI API
- **규모/일정**: 소형 MVP, 2개월
- **예산 범위**: M/M 기반 자동 산출
- **제안서 톤**: 밸런스형

## Part 1: 현황 분석 및 요구사항 정의

### 1.1 프로젝트 배경 및 목적

**추진 배경**
- 건설/제조업에서 발생하는 잉여자재, 불용자재의 비효율적 처리
- 기존 쇼핑몰형 플랫폼의 한계: 개별 등록 번거로움, 수동 검색의 비효율
- 판매자는 대량 자재 등록에 시간 소요, 구매자는 원하는 자재 찾기 어려움

**해결하려는 문제**
1. 대량 자재 등록의 번거로움 → 엑셀 일괄 업로드로 해결
2. 수동 검색의 비효율 → AI가 자동으로 매칭 후 먼저 제안
3. 복잡한 가입 절차 → OCR로 사업자등록증 자동 인식
4. 복잡한 결제 프로세스 → 직접 송금으로 간소화

**기대 목표**
- 자재 등록 시간 80% 단축 (개별 등록 → 엑셀 일괄)
- 매칭 성공률 60% 향상 (수동 검색 → AI 자동 제안)
- 거래 성사 시간 50% 단축 (PG 결제 → 직접 송금)

### 1.2 현황 분석

**유사 서비스 비교표**

| 구분 | 기존 쇼핑몰형 플랫폼 | **제안 AI 매칭형** | 차별점 |
|------|------------------|-----------------|-------|
| **등록** | 개별 등록 (이미지 필수) | 엑셀 일괄 업로드 (텍스트 위주) | 대량 등록 용이 |
| **탐색** | 고객 직접 검색 (Passive) | AI 분석 후 먼저 제안 (Active) | 능동적 매칭 |
| **가입** | 수기 입력 | OCR 자동 인식 | 가입 시간 단축 |
| **거래** | PG 결제 (복잡) | 직접 송금 (신속) | 수수료 절감 |
| **신뢰도** | 리뷰/평점 | AI 매칭 점수 + 거래 이력 | 데이터 기반 신뢰 |

**시장 현황**
- B2B 자재 거래 시장은 여전히 전화/이메일 기반 오프라인 중심
- 기존 온라인 플랫폼은 쇼핑몰 방식으로 B2C 경험을 그대로 적용
- AI 기반 능동적 매칭 서비스는 시장에 아직 없음 (선점 기회)

### 1.4 비기능 요구사항 (NFR)

**성능**
- API 응답 속도: 평균 300ms 이하
- AI 매칭 처리: 10초 이내 (배치 처리)
- 엑셀 업로드: 최대 1,000건 동시 처리

**보안**

- JWT 기반 인증/인가

- OWASP Top 10 보안 취약점 대응

**가용성**
- SLA 목표: 99.5% (MVP 기준)

**확장성**

#### 주요 사용자 흐름 (User Flow)

**Flow 1: 판매자 - 자재 등록 및 매칭**
```
회원가입 (OCR) → 로그인 → 엑셀 템플릿 다운로드 → 자재 정보 입력
→ 엑셀 업로드 → AI 자동 매칭 시작 → 매칭 발견 시 알림
→ 구매자 거래 요청 수신 → 승인 → 연락처 공유 → 거래 완료
```

**Flow 2: 구매자 - 매칭 결과 확인 및 거래**
```
회원가입 (OCR) → 로그인 → AI 매칭 결과 조회
→ 추천 자재 확인 → 상세 보기 → 거래 요청
→ 판매자 승인 대기 → 승인 시 연락처 확인 → 거래 완료
```

**Flow 3: AI 매칭 엔진 (백그라운드)**
```
[배치 작업 - 10분마다 실행]
판매 자재 목록 조회 → 텍스트 임베딩 생성 (OpenAI)
→ 구매 니즈 목록 조회 → 벡터 유사도 계산
→ 매칭 점수 70% 이상 → 알림 발송
```

### 2.5 데이터 모델 개요

#### 핵심 엔티티 관계도 (ERD)

```
┌─────────────────┐         ┌─────────────────┐
│     users       │         │   materials     │
├─────────────────┤         ├─────────────────┤
│ id (PK)         │─────────│ id (PK)         │
│ email           │   1:N   │ seller_id (FK)  │
│ company_name    │         │ name            │
│ business_no     │         │ category        │
│ role (enum)     │         │ quantity        │
│ phone           │         │ unit_price      │
│ created_at      │         │ location        │
└─────────────────┘         │ description     │
                            │ embedding       │
                            │ status          │
                            │ created_at      │
                            └─────────────────┘
                                     │
                                     │ 1:N
                                     ▼
┌─────────────────┐         ┌─────────────────┐
│  transactions   │         │   matchings     │
├─────────────────┤         ├─────────────────┤
│ id (PK)         │         │ id (PK)         │
│ material_id (FK)│◄────────│ material_id (FK)│
│ buyer_id (FK)   │   1:1   │ buyer_id (FK)   │
│ seller_id (FK)  │         │ score           │
│ status (enum)   │         │ reason          │
│ request_at      │         │ created_at      │
│ approved_at     │         └─────────────────┘
│ completed_at    │
└─────────────────┘

enum role: seller | buyer
enum status (materials): active | matched | sold
enum status (transactions): requested | approved | completed | cancelled
```

#### 주요 테이블 설명

**users** (회원)
- 판매자/구매자 구분 (role)
- OCR로 추출한 사업자등록번호, 회사명 저장

**materials** (자재)
- 판매자가 등록한 자재 정보
- embedding: OpenAI 텍스트 임베딩 (벡터 검색용)
- status: active(판매중) / matched(매칭됨) / sold(판매완료)

**matchings** (매칭 결과)
- AI가 생성한 매칭 결과
- score: 매칭 점수 (0~100)
- reason: 매칭 사유 (AI 생성 텍스트)

**transactions** (거래)
- 구매자 거래 요청 → 판매자 승인 → 거래 완료 흐름
- status: requested → approved → completed

## 핵심 차별점 요약

1. **엑셀 일괄 업로드** — 대량 자재 등록 시간 80% 단축
2. **AI 능동적 매칭** — 판매자/구매자가 찾아다니지 않아도 자동 제안
3. **OCR 자동 인식** — 가입 시간 5분 → 1분 단축
4. **직접 송금** — PG 수수료 없음, 거래 즉시 가능

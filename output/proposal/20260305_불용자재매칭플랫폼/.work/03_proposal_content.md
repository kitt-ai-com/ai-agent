# 제안서 본문 및 UI/UX 문서

## 사전 브리핑
- **프로젝트명**: 불용자재 AI 매칭 플랫폼
- **제안서 톤**: 밸런스형 (기술과 비즈니스 가치 균형)
- **핵심 메시지**: AI 기반 능동적 매칭으로 거래 효율 극대화

---

## Part 1: 제안 개요

### 1.1 제안 배경 및 목적

**고객의 비즈니스 과제**
- 건설/제조업체에서 발생하는 불용자재(잉여자재)는 연간 수조 원 규모
- 기존 처리 방식: 폐기(손실) 또는 개별 연락(시간 소요)
- 온라인 플랫폼 존재하나 쇼핑몰 방식으로 B2B 특성 미반영
  - 판매자: 자재 100개 등록 시 개별 입력으로 5시간 소요
  - 구매자: 원하는 자재 찾기 위해 수십 페이지 검색 필요

**본 프로젝트의 해결 방안**
- **AI 능동적 매칭**: 구매자가 찾아다니지 않아도 AI가 먼저 제안
- **엑셀 일괄 업로드**: 대량 자재 등록 시간 80% 단축 (5시간 → 1시간)
- **OCR 자동 인식**: 가입 시간 5분 → 1분 단축
- **직접 송금**: PG 수수료 없음, 거래 즉시 가능

**핵심 가치 제안 (Value Proposition)**

> "찾는 것이 아니라 찾아주는 불용자재 거래 플랫폼"
> - 판매자는 엑셀 한 번에 대량 등록
> - 구매자는 AI가 자동으로 찾아주는 자재만 확인
> - 거래 성사 시간 50% 단축, 수수료 0원

### 1.3 프로젝트 핵심 차별점

#### 경쟁사 대비 우리의 강점 3가지

| 차별점 | 기존 플랫폼 | **본 제안** | 효과 |
|--------|-----------|-----------|------|
| **1. 등록 방식** | 개별 등록 (이미지 필수) | 엑셀 일괄 업로드 (텍스트 위주) | 등록 시간 80% 단축 |
| **2. 매칭 방식** | 고객 직접 검색 (Passive) | AI 분석 후 먼저 제안 (Active) | 매칭 성공률 60% 향상 |
| **3. 거래 방식** | PG 결제 (수수료 3~5%) | 직접 송금 (수수료 0%) | 거래 비용 절감 |

**기술적 강점**
- **Next.js 14 App Router**: 최신 React 서버 컴포넌트로 빠른 로딩
- **Supabase**: PostgreSQL + Auth + Storage 통합으로 개발 속도 2배
- **OpenAI GPT-4 Embedding**: 텍스트 기반 정확한 자재 매칭 (70% 이상 정확도)
- **CLOVA OCR**: 한국 사업자등록증 인식률 95% 이상

**비즈니스 강점**
- **빠른 MVP 출시**: 2개월 내 시장 검증 가능
- **확장 용이**: Phase 2/3로 점진적 기능 확대
- **운영 비용 절감**: 서버리스 아키텍처로 초기 인프라 비용 최소화

## Part 2: UI/UX 설계 방향

### 2.1 UX 설계 원칙

#### 핵심 설계 원칙 5가지

1. **간결함 (Simplicity)**
   - B2B 사용자는 빠른 업무 처리를 원함
   - 불필요한 클릭 최소화, 한 화면에 핵심 정보 집중

2. **자동화 (Automation)**
   - 수동 입력 최소화: 엑셀 업로드, OCR, AI 매칭
   - 사용자는 확인/승인만 하면 됨

3. **신뢰성 (Trust)**
   - 매칭 점수와 사유를 투명하게 제시
   - 사업자등록증 인증으로 신뢰도 확보

4. **효율성 (Efficiency)**
   - 대량 데이터 처리 최적화 (엑셀 1,000건 10초 이내)
   - 로딩 최소화 (Skeleton UI, Lazy Loading)

5. **접근성 (Accessibility)**
   - 중장년층 사용자 고려 (큰 폰트, 명확한 버튼)
   - 모바일 반응형 (외부 현장에서도 사용)

#### 핵심 사용자 시나리오 (주요 3개 플로우)

**시나리오 1: 판매자 - 최초 자재 등록**
```
1. 회원가입 → 사업자등록증 사진 촬영 → OCR 자동 인식 → 정보 확인
2. 로그인 → [자재 등록] 클릭 → 엑셀 템플릿 다운로드
3. 엑셀에 자재 목록 입력 (이름, 수량, 가격, 위치, 상태)
4. 엑셀 파일 업로드 → AI가 자동 분석 → "100건 등록 완료" 알림
5. 대시보드에서 매칭 현황 실시간 확인
```

**시나리오 2: 구매자 - AI 추천 자재 확인 및 거래**
```
1. 회원가입 → 로그인
2. [AI 매칭 결과] 화면 → "5건의 추천 자재가 있습니다" 알림
3. 자재 목록 확인 (매칭 점수 순 정렬)
4. 자재 상세 보기 → 매칭 사유 확인 ("귀사가 찾던 H빔 200x200과 일치")
5. [거래 요청] 버튼 클릭 → 판매자 승인 대기
6. 승인 시 연락처 확인 → 직접 연락 → 거래 완료 처리
```

**시나리오 3: AI 매칭 엔진 - 자동 매칭**
```
[배치 작업 - 10분마다 자동 실행]
1. 판매 자재 목록 조회 (status = 'active')
2. 각 자재의 텍스트 임베딩 생성 (OpenAI API)
3. 구매 니즈 목록 조회 (구매자가 과거 검색한 키워드 또는 관심 자재)
4. 벡터 유사도 계산 (코사인 유사도)
5. 매칭 점수 70% 이상 → matchings 테이블에 저장
6. 구매자에게 이메일/푸시 알림 발송
```

### 2.2 UI 디자인 방향

#### 디자인 컨셉 키워드

- **신뢰감** (Trust): 전문적인 B2B 플랫폼 이미지
- **직관적** (Intuitive): 설명 없이도 사용 가능한 인터페이스
- **효율적** (Efficient): 불필요한 장식 없이 정보 중심

#### 톤앤매너

**컬러 방향**
- **Primary**: 블루 계열 (#1E40AF) - 신뢰감, 전문성
- **Secondary**: 그린 계열 (#10B981) - 거래 성공, 긍정적 액션
- **Accent**: 오렌지 (#F97316) - 알림, 중요 정보
- **Background**: 화이트/라이트 그레이 - 가독성 최우선

**타이포그래피**
- **헤딩**: Pretendard Bold (한글), Inter Bold (영문)
- **본문**: Pretendard Regular (한글), Inter Regular (영문)
- **폰트 크기**: 최소 14px (중장년층 고려)

**반응형 전략**
- **Desktop First**: 주 사용 환경은 사무실 PC
- **Mobile Responsive**: 외부 현장에서도 사용 가능하도록 최적화
- **Breakpoint**: 1024px (Desktop) / 768px (Tablet) / 375px (Mobile)

### 5.3 투자 대비 기대 가치 (ROI)

**초기 투자**
- 개발 비용: 6,100만원 (부가세 포함)
- 연간 유지보수: 3,200만원
- **총 1년 투자**: 9,300만원

**예상 수익 (오픈 후 1년 기준)**
- 월간 거래 건수: 500건 (보수적 추정)
- 거래당 수수료: 1% (직접 송금이지만 플랫폼 이용료로 징수 가능)

- **월간 수익**: 500건 × 200만원 × 1% = 1,000만원
- **연간 수익**: 1,000만원 × 12개월 = **1.2억원**

**ROI**
- (연간 수익 1.2억 - 총 투자 0.93억) / 총 투자 0.93억 × 100 = **약 29% ROI**
- 1년 3개월 만에 투자 회수 가능

## 1. 제안 개요

### 1.1 제안 배경 및 목적

**비즈니스 과제**
- 건설/제조업에서 발생하는 불용자재(잉여자재)는 연간 수조 원 규모
- 기존 처리 방식: 폐기(손실) 또는 개별 연락(시간 소요)
- 온라인 플랫폼은 쇼핑몰 방식으로 B2B 특성 미반영
  - **판매자**: 자재 100개 등록 시 개별 입력으로 **5시간** 소요
  - **구매자**: 원하는 자재 찾기 위해 수십 페이지 검색 필요

**해결 방안**

| 문제 | 해결책 | 효과 |
|------|--------|------|
| 대량 등록 번거로움 | **엑셀 일괄 업로드** | 등록 시간 **80% 단축** |
| 수동 검색 비효율 | **AI 능동적 매칭** | 매칭 성공률 **60% 향상** |
| 복잡한 가입 절차 | **OCR 자동 인식** | 가입 시간 **5분 → 1분** |
| PG 수수료 부담 | **직접 송금** | 수수료 **0%** |

**핵심 가치 제안**
> "찾는 것이 아니라 찾아주는 불용자재 거래 플랫폼"
> - 판매자는 엑셀 한 번에 대량 등록
> - 구매자는 AI가 자동으로 찾아주는 자재만 확인
> - 거래 성사 시간 50% 단축, 수수료 0원

### 1.2 제안 범위

#### ✅ 포함 범위

- **핵심 기능**: 회원가입(OCR), 엑셀 일괄 업로드, AI 자동 매칭, 거래 관리
- **화면 개발**: 총 8개 화면 (회원 2개, 판매자 2개, 구매자 2개, 공통 2개)
- **기술 스택**: Next.js 14 + Supabase + OpenAI API + CLOVA OCR
- **인프라**: Vercel + Supabase Cloud (프로덕션 배포 포함)
- **산출물**: 소스 코드, API 문서, 운영 가이드, 사용자 매뉴얼

#### ❌ 제외 범위 (Phase 2 고려)

- 고급 검색/필터, 이미지 인식, 실시간 채팅, 에스크로 결제, 분석 리포트, 모바일 앱

### 1.3 핵심 차별점

| 구분 | 기존 플랫폼 | **본 제안** | 효과 |
|------|-----------|-----------|------|
| **등록** | 개별 등록 (이미지 필수) | 엑셀 일괄 업로드 | 등록 시간 **80% 단축** |
| **탐색** | 고객 직접 검색 (Passive) | AI 분석 후 먼저 제안 (Active) | 매칭 성공률 **60% 향상** |
| **가입** | 수기 입력 | OCR 자동 인식 | 가입 시간 **5분 → 1분** |
| **거래** | PG 결제 (수수료 3~5%) | 직접 송금 (수수료 0%) | 수수료 **100% 절감** |

## 2. 현황 분석 및 요구사항

### 2.1 시장 현황

- B2B 자재 거래 시장은 여전히 전화/이메일 기반 오프라인 중심
- 기존 온라인 플랫폼은 쇼핑몰 방식으로 B2C 경험을 그대로 적용
- **AI 기반 능동적 매칭 서비스는 시장에 아직 없음** (선점 기회)

### 2.2 유사 서비스 비교

| 구분 | 기존 쇼핑몰형 | **제안 AI 매칭형** | 차별점 |
|------|-------------|-----------------|-------|
| 등록 | 개별 등록 (이미지 필수) | 엑셀 일괄 업로드 | 대량 등록 용이 |
| 탐색 | 고객 직접 검색 | AI 분석 후 먼저 제안 | 능동적 매칭 |
| 가입 | 수기 입력 | OCR 자동 인식 | 가입 시간 단축 |
| 거래 | PG 결제 | 직접 송금 | 수수료 절감 |
| 신뢰 | 리뷰/평점 | AI 매칭 점수 + 거래 이력 | 데이터 기반 신뢰 |

### 2.3 기능 요구사항 (FR)

#### MVP 핵심 기능 (8개 화면)

| 화면 ID | 화면명 | 주요 기능 | 우선순위 |
|---------|--------|----------|---------|
| S-001 | 회원가입 | OCR 사업자등록증 인식, 기업 정보 입력 | 필수 |
| S-002 | 로그인 | 이메일/소셜 로그인, JWT 토큰 발급 | 필수 |
| S-101 | 자재 등록 (엑셀) | 엑셀 파일 업로드, 템플릿 다운로드, 검증 | 필수 |
| S-102 | 내 자재 목록 | 등록 자재 조회, 수정, 삭제, 매칭 현황 | 필수 |
| S-201 | AI 매칭 결과 | 추천 자재 목록, 매칭 점수, 필터 | 필수 |
| S-202 | 자재 상세 | 자재 정보, 가격, 수량, 위치, 거래 요청 | 필수 |
| S-301 | 거래 관리 | 거래 요청/승인, 연락처 공유, 거래 완료 | 필수 |
| S-302 | 내 정보 | 기업 정보 수정, 비밀번호 변경, 알림 설정 | 필수 |

### 2.4 비기능 요구사항 (NFR)

- **성능**: API 응답 300ms 이하, AI 매칭 10초 이내, 동시 접속 500명
- **보안**: 사업자등록증 암호화, JWT 인증, HTTPS, OWASP Top 10 대응
- **가용성**: SLA 99.5%, 매일 백업, 장애 복구 24시간 이내
- **확장성**: Vercel 자동 확장, Supabase 수평 확장

### 3.2 기술 스택

#### 프론트엔드
- **Next.js 14**: SSR/SSG, SEO 최적화, API Routes 통합
- **React 18**: 컴포넌트 기반 개발
- **TypeScript**: 타입 안정성
- **TailwindCSS**: 빠른 스타일링
- **Shadcn/ui**: 고품질 UI 컴포넌트

#### 백엔드
- **Next.js API Routes**: 서버리스 API
- **Supabase**: PostgreSQL + Auth + Storage 통합
- **Prisma**: 타입 안전 ORM

#### AI/ML
- **OpenAI API (GPT-4)**: 텍스트 임베딩, 자재 매칭
- **Langchain**: AI 워크플로우
- **CLOVA OCR**: 한국 사업자등록증 인식

#### 인프라
- **Vercel**: Next.js 최적화 배포
- **Supabase Cloud**: 데이터베이스 관리
- **GitHub Actions**: CI/CD 자동화
- **Sentry**: 에러 모니터링

### 3.3 핵심 기능 상세

#### 1) 엑셀 일괄 업로드

```
1. 판매자가 [엑셀 템플릿 다운로드] 클릭
2. 템플릿에 자재 정보 입력 (이름, 수량, 가격, 위치, 상태)
3. 엑셀 파일 업로드
4. 서버에서 검증 (필수 항목, 형식, 중복)
5. ✅ 95건 성공 / ⚠️ 5건 오류 → 상세 오류 메시지 표시
6. 등록 완료 → AI 매칭 엔진 자동 실행
```

#### 2) AI 자동 매칭 엔진

```
[배치 작업 - 10분마다 자동 실행]
1. 판매 자재 목록 조회 (status = 'active')
2. 각 자재의 텍스트 임베딩 생성 (OpenAI Embedding API)
3. 구매 니즈 목록 조회 (구매자가 과거 검색한 키워드)
4. 벡터 유사도 계산 (코사인 유사도)
5. 매칭 점수 70% 이상 → matchings 테이블에 저장
6. 구매자에게 이메일/푸시 알림 발송
```

#### 3) OCR 자동 인식

```
1. 회원가입 시 [사업자등록증 촬영] 클릭
2. 카메라/갤러리에서 이미지 선택
3. CLOVA OCR API 호출
4. 사업자번호, 회사명, 대표자명 자동 추출
5. 사용자가 확인/수정
6. 회원가입 완료 (1분 이내)
```

## 4. UI/UX 설계 방향

### 4.1 UX 설계 원칙

1. **간결함**: 불필요한 클릭 최소화, 한 화면에 핵심 정보 집중
2. **자동화**: 수동 입력 최소화 (엑셀, OCR, AI 매칭)
3. **신뢰성**: 매칭 점수와 사유를 투명하게 제시
4. **효율성**: 대량 데이터 처리 최적화 (엑셀 1,000건 10초 이내)
5. **접근성**: 중장년층 고려 (큰 폰트 14px+, 명확한 버튼)

### 4.2 디자인 컨셉

- **컬러**: Primary 블루(#1E40AF), Secondary 그린(#10B981), Accent 오렌지(#F97316)
- **타이포그래피**: Pretendard (한글), Inter (영문), 최소 14px
- **반응형**: Desktop First (1024px+), Mobile Responsive (375px+)

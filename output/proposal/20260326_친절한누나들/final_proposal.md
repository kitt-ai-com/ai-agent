## 1. 제안 개요

### 1.1 제안 배경 및 목적

**친절한누나들**은 경기도 조합을 기반으로 3,000명 이상의 딜러 네트워크를 보유하고 있으며, 자동차 구매 DB를 하루 150~200건씩 확보하고 있습니다. 그러나 회의록 분석 결과, **"실행이 안 되고 있는 상황"**이 가장 큰 과제로 확인되었습니다.

**핵심 과제**
1. **실행력 부재**: DB는 쌓이지만 마케팅 수단이 없음 (오프라인 전단지/문자 발송 수준)
2. **딜러 리텐션 미흡**: 3,000명 네트워크 중 3개월 미이용 딜러 대량 발생
3. **수익 구조 단순화**: DB 판매 단일 수익원, 보험 중개 미활용
4. **고객 LTV 미활용**: 차량 교체 주기(3~7년) 기반 CRM 부재

본 플랫폼은 이러한 과제를 해결하기 위해 **5~6개월 내 빠른 실행**이 가능한 자동화 솔루션을 제안합니다.

### 1.2 제안 범위

**Phase 1 (필수 기능)**
- 딜러 관리 + 등급제 (Gold/Silver/Bronze)
- DB 자동 배분 + 48시간 룰
- 카카오 알림톡 자동화 (템플릿 10개+)
- 실시간 대시보드
- 고객 DB 관리 + 차량 라이프사이클 추적
- 보험 비교 및 중개 (3개 이상 보험사)

**Phase 2 (확장 기능)**
- 복지몰 연동 (싼타몰)
- B2C 라이프 파트너 (정비소 제휴)
- 실시간 채팅
- 금융 중개 (캐피탈/리스)

### 1.3 핵심 가치 제안

1. **실행력 확보**: 마케팅 자동화 도구로 "실행 안 됨" 문제 해결, 72시간 룰 적용으로 전환율 2~3배 향상
2. **딜러 리텐션**: 등급제 + 자동 배분 + 알림톡으로 딜러 활성률 40% → 70% 개선 (30%p 증가)
3. **양면 수익화**: 딜러 15% + 고객 85% 수익 구조로 보험 중개 메인화, 월 수익 $50,000+ 달성

## 2. 현황 분석 및 요구사항

### 2.1 시장/서비스 현황 분석

#### 2.1.1 시장 트렌드: 양면 수익화 모델

해외 자동차 플랫폼은 **양면 수익화 모델**을 채택하여, 딜러(공급자) 15% + 고객(소비자) 85%의 수익 균형을 유지합니다. 특히 **보험 중개**는 가장 높은 마진(15~20%)을 제공하며, 단일 수익원으로 월 $50,000 이상 달성이 가능합니다.

**수익 구조 비교**

| 수익원 | 마진율 | 딜러 기여 | 고객 기여 | 비고 |
|--------|--------|-----------|-----------|------|
| 보험 중개 | 15~20% | 5% | 95% | **최고 마진, 메인 수익원** |
| 금융 중개 (캐피탈/리스) | 3~5% | 20% | 80% | 거래당 수수료 |
| 거래 수수료 | 2~3% | 30% | 70% | 차량 매매 완료 시 |
| 딜러 구독료 | - | 100% | - | 월 $30~100 |
| 리드 수수료 | - | 100% | - | 리드당 $50~100 |
| 부가 서비스 (정비/보험) | 10~15% | 0% | 100% | B2C 직접 제공 |

**합산**: 딜러 15% + 고객 85% = **안정적 양면 수익**

### 3.1 시스템 아키텍처

```

│  - 대시보드           - 리드 확인          - 보험 비교           │
│  - 딜러 관리          - 성과 리포트        - 견적 신청           │

                    │ HTTPS (SSL/TLS)
┌───────────────────▼─────────────────────────────────────────────┐

└───────────────────┬─────────────────────────────────────────────┘
                    │ REST API / GraphQL
┌───────────────────▼─────────────────────────────────────────────┐
│                  백엔드 계층 (Supabase BaaS)                      │

│  [Storage] 파일 저장소 (프로필 이미지, 차량 사진)                  │

                    │ Webhook / API

├─────────────────────────────────────────────────────────────────┤
```

**아키텍처 특징**
- **서버리스 우선**: Supabase + Vercel로 인프라 관리 부담 최소화
- **보안**: RLS (Row Level Security)로 데이터 접근 제어 자동화
- **확장성**: Auto-scaling으로 트래픽 증가 시 자동 대응
- **비용 효율**: BaaS 활용으로 초기 인프라 비용 90% 절감

**주요 테이블 설명**

1. **dealers**: 딜러 정보, 등급 (Gold/Silver/Bronze), 마지막 활동 시각
2. **leads**: 자동차 구매 DB, 배분 상태 (미처리/진행중/완료), 딜러 연결
3. **customers**: 고객 정보, 성별/주소/연락처
4. **vehicles**: 차량 이력, 구매 시점, 교체 예측일
5. **insurance_policies**: 보험 가입 내역, 중개 수수료 (15~20%)
6. **insurance_quotes**: 보험 견적 이력 (3개 보험사 비교)
7. **grade_history**: 딜러 등급 변경 이력 (Gold → Silver 등)
8. **kakao_messages**: 알림톡 발송 이력, 오픈율 추적 (sent_at → opened_at)
9. **users** (미표시): 통합 인증 테이블 (관리자/딜러/고객)
10. **n8n_workflows** (미표시): 자동화 워크플로우 설정

**RLS (Row Level Security) 정책**
- 딜러는 자신에게 배분된 리드만 조회 가능
- 고객은 자신의 정보만 조회/수정 가능
- 관리자는 모든 데이터 접근 가능

## 4. UI/UX 설계 방향

### 4.1 UX 설계 원칙 (5개)

1. **모바일 우선 (Mobile-First)**: 딜러/고객은 모바일 접속 비율 90%+ → 모바일 화면 최적화 우선
2. **3 Tap 원칙**: 주요 기능은 3번 탭 이내 접근 (예: 홈 → 리드 목록 → 리드 상세)
3. **실시간 피드백**: 알림톡 발송, DB 배분 등 모든 액션에 즉시 피드백 제공 (토스트 알림)
4. **접근성 (WCAG 2.1 AA)**: 색맹 대응 (색상만으로 정보 전달 금지), 키보드 네비게이션 지원
5. **일관성**: shadcn/ui 디자인 시스템으로 모든 화면 일관된 버튼/폼/테이블 스타일

### 4.2 UI 디자인 컨셉

**컬러 시스템**
- **Primary**: 카카오 옐로우 (#FEE500) → 브랜드 연상, 알림톡 연계
- **Secondary**: 네이비 (#1E3A8A) → 신뢰감, 관리자 화면
- **Success**: 그린 (#10B981) → 전환 완료, 등급 상승
- **Warning**: 오렌지 (#F59E0B) → 48시간 룰 임박, 미처리 DB
- **Error**: 레드 (#EF4444) → 실패, 등급 하락

**타이포그래피**
- **Heading**: Pretendard Bold (한글), Inter Bold (영문)
- **Body**: Pretendard Regular (한글), Inter Regular (영문)
- **Size**: 모바일 기준 16px (본문), 24px (제목), 14px (설명)

**아이콘**
- Lucide Icons (오픈소스) → shadcn/ui 기본 제공

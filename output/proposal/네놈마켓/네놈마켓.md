## 1. 프로젝트 개요

### 1.1 서비스 소개

**네놈마켓**은 농산물, 수산물, 축산물 B2B 도매/유통을 위한 차세대 플랫폼입니다.

기존 B2B 유통 업무의 비효율성을 해결하고, **AI 채팅 인터페이스**를 통해 자연어로 모든 업무를 처리할 수 있는 혁신적인 솔루션을 제공합니다.

### 1.2 핵심 가치 제안

✨ **AI 채팅으로 업무 자동화**
- "오늘 주문 현황 알려줘", "발주서 뽑아줘" 같은 자연어 명령으로 즉시 처리
- 엑셀 파일 업로드만으로 자동 주문 등록 및 송장 매칭

⚡ **실시간 가격 관리**
- 가격 변동이 잦은 신선 품목에 최적화
- 등급별, 셀러별 차등 가격 자동 적용

🔄 **통합 주문 관리**
- 쿠팡, 네이버 스마트스토어 등 다채널 주문 통합
- 공급사별 발주서 자동 생성
- 송장 자동 매칭 및 배송 추적

📊 **스마트 정산 시스템**
- 등급별/개별 수수료율 자동 계산
- 실시간 매출/통계 대시보드
- 클레임 자동 반영

### 1.3 타겟 사용자

| 역할 | 설명 | 주요 니즈 |
|------|------|-----------|
| **플랫폼 관리자** | 네놈마켓 운영자 | 효율적인 전체 운영 관리, 정확한 정산, 공급사 커뮤니케이션 자동화 |
| **셀러** | 쿠팡/스마트스토어 판매자 | 빠른 주문 처리, 실시간 송장 확인, 간편한 상품 조회 |
| **공급사** | 상품 공급 업체 | 정확한 발주서 수신, 간편한 송장 회신 |

## 2. 제안 배경 및 목적

### 2.1 현재 B2B 유통 시장의 문제점

#### 📂 엑셀 기반 수작업의 한계
- 주문 데이터 수동 입력/정리로 인한 오류 발생
- 채널별 주문서 형식이 달라 통합 관리 어려움
- 송장 매칭을 수작업으로 처리 → 시간 소모 및 누락 위험

#### 💰 실시간 가격 변동 대응 어려움
- 농수산물 특성상 일일 가격 변동 빈번
- 엑셀로 개별 수정 시 실수 및 지연 발생
- 셀러별 차등 가격 적용이 복잡

#### 📞 비효율적인 공급사 커뮤니케이션
- 카카오톡으로 발주서 전달 → 양식 불일치
- 송장 회신 지연 및 누락
- 발주/입고 이력 추적 어려움

#### 📊 정산 및 통계의 불투명성
- 수동 정산으로 인한 오류
- 실시간 매출 현황 파악 어려움
- 클레임 처리 시 정산 반영 복잡

### 2.2 네놈마켓의 솔루션

#### 🤖 AI 채팅으로 모든 업무 자동화
```
관리자: "CJ공급사 발주서 뽑아줘"
AI: 📎 CJ공급사_발주서_20260303.xlsx (15건, 총 2,340,000원)

셀러: "오늘 올린 주문 중 송장 들어온 거 있어?"
AI: 15건 중 12건 송장 입력 완료. 3건 대기 중.
    📎 송장입력완료_20260303.xlsx
```

#### ⚡ 실시간 스마트 가격 관리
- 등급별 자동 가격 적용 (VIP 5% 할인, 일반 정가 등)
- 특정 셀러별 개별 가격 설정 가능
- "감귤류 전체 10% 인상해줘" 같은 일괄 변경 지원

#### 🔄 자동 주문 처리 시스템
1. 셀러가 어떤 형식의 엑셀이든 업로드
2. AI가 자동으로 컬럼 인식 (고객명, 주소, 상품, 수량 등)
3. 주문 자동 등록 및 공급사별 취합
4. 발주서 자동 생성 (공급사 양식 맞춤)
5. 송장 업로드 시 자동 매칭

#### 📊 투명한 정산 시스템
- 등급별/개별 수수료율 자동 계산
- 반품/환불 자동 반영
- 실시간 매출 통계 대시보드
- 일별/주별/월별 비교 차트

## 3. 핵심 기능

### 3.1 플랫폼 관리자 기능

#### 📦 상품 관리
- ✅ **개별/대량 상품 등록**: 수동 입력 또는 엑셀 업로드
- ✅ **등급별 차등 가격 설정**: VIP, 골드, 일반 등 등급별 가격
- ✅ **셀러별 개별 가격 설정**: 특정 셀러에게만 특별 가격 적용
- ✅ **실시간 가격 수정**: 개별 또는 카테고리 일괄 변경
- ✅ **재고 관리**: 수량 추적 및 품절 자동 처리
- 🔜 **재고 부족 알림**: 설정 수량 이하 시 자동 알림

#### 📋 주문 관리
- ✅ **셀러 주문 접수**: 엑셀 업로드 또는 수동 입력
- ✅ **주문 취합**: 공급사별/상품별 자동 취합
- ✅ **발주서 자동 생성**: 공급사 양식에 맞춘 엑셀 다운로드
- 🔜 **송장 자동 매칭**: 공급사 송장 엑셀 업로드 시 주문별 자동 매칭
- ✅ **주문 상태 관리**: 준비중 → 배송중 → 배송완료

#### 🚚 배송 관리
- 🔜 **택배 API 연동**: CJ대한통운 API로 자동 집하 요청
- ✅ **송장 일괄/개별 입력**: 엑셀 업로드 또는 수동 입력
- 🔜 **배송 추적**: 실시간 배송 현황 조회
- ✅ **송장 알림**: 셀러에게 자동 알림

#### 👥 회원 관리
- ✅ **셀러 가입 승인**: 사업자등록증 확인 후 승인/반려
- ✅ **등급 시스템**: 등급 생성/수정/삭제, 등급별 수수료율 설정
- ✅ **개별 수수료 설정**: 특정 셀러에게 별도 수수료율 적용
- ✅ **셀러 정보 관리**: 연락처, 사업자 정보 등

#### 💰 정산/매출 관리
- 🔜 **자동 정산**: 매출 집계 및 수수료 자동 계산
- 🔜 **클레임 반영**: 반품/환불 금액 정산 자동 반영
- 🔜 **통계 대시보드**:
  - 일별/주별/월별 매출 통계 (총매출, 순매출)
  - 상품별/카테고리별 판매량
  - 셀러별 주문수/매출/수수료
  - 상품별 판매량/매출 Top N
  - 주문 상태별 현황
  - 기간별 비교 차트
- 🔜 **정산 내역 엑셀 다운로드**

#### 🛠️ C/S 관리
- 🔜 **문의 관리**: 게시판 문의 확인 및 답변
- 🔜 **클레임 처리**:
  - 반품/환불/교환 요청 접수
  - 승인/거부 처리
  - 정산 자동 반영
  - 교환 시 재배송 처리

#### 🏭 공급사 관리
- ✅ **공급사 정보 등록**: 업체명, 연락처, 담당자 등
- 🔜 **발주 양식 관리**: 공급사별 맞춤 엑셀 템플릿
- ✅ **발주/입고 이력**: 전체 이력 조회 및 관리

#### 🔔 알림/공지
- 🔜 **셀러 공지사항**: 전체/개별 공지 발송
- 🔜 **가격 변동 안내**: 상품 가격 변경 시 자동 알림
- 🔜 **카카오 알림톡**: 주요 이벤트 알림

#### 🛍️ 상품 조회

### 3.3 AI 채팅 인터페이스 (핵심 차별화 요소)

#### 🤖 자연어 명령 처리

**관리자 명령 예시**
```
"오늘 주문 현황 알려줘"
"CJ공급사 발주서 뽑아줘"
"감귤류 전체 10% 인상해줘"
"김셀러 이번 달 정산 내역 보여줘"
"지난 주 가장 많이 팔린 상품 Top 5"
"전체 셀러에게 공지 보내줘: 내일 감귤 가격 인상"
```

**셀러 명령 예시**
```
"주문 업로드할게" + 📎쿠팡_주문.xlsx
"오늘 올린 주문 중 송장 들어온 거 있어?"
"홍길동 고객 주문 정보 알려줘"
"송장번호 1234567890 주문 보여줘"
"이번 주 올린 주문 전체 송장 파일 다운로드"
"제주감귤 재고 확인"
```

#### 📎 파일 처리
- **업로드**: 엑셀 파일 첨부 → AI가 자동 인식 및 처리
- **다운로드**: 발주서, 송장 파일, 정산 내역 등 즉시 생성
- **자동 매핑**: 어떤 형식의 엑셀이든 컬럼 자동 인식

#### 🔔 자동 알림
- 새 주문 접수 시
- 송장 입력 완료 시
- 재고 부족 시
- 가격 변동 시
- 클레임 접수 시

#### 🎯 역할별 권한 제어
- 관리자: 모든 기능 접근 가능
- 셀러: 본인 주문만 조회, 상품 등록/수정 불가

## 4. 비즈니스 플로우

### 4.1 공급사 상품 주문 → 배송 플로우

```
1️⃣ 셀러: 주문 엑셀 업로드 (쿠팡/스마트스토어 등)
    ↓
2️⃣ AI: 자동 컬럼 인식 → 주문 등록
    ↓
3️⃣ 관리자: "CJ공급사 발주서 뽑아줘"
    ↓
4️⃣ AI: 공급사 양식 발주서 자동 생성 + 다운로드
    ↓
5️⃣ 관리자: 카톡으로 공급사에게 발주서 전달
    ↓
6️⃣ 공급사: 송장 엑셀 회신 (카톡)
    ↓
7️⃣ 관리자: 송장 엑셀 업로드
    ↓
8️⃣ AI: 주문별 자동 매칭 → 상태 "배송중" 변경
    ↓
9️⃣ 셀러: 송장 자동 반영 확인 → 자기 채널에 입력
```

### 4.2 자체 상품 주문 → 배송 플로우 (택배 API 연동)

```
1️⃣ 셀러: 주문 엑셀 업로드
    ↓
2️⃣ AI: 자동 주문 등록
    ↓
3️⃣ 관리자: "자체 상품 배송 집하 요청"
    ↓
4️⃣ 시스템: 택배사 API 자동 집하 요청
    ↓
5️⃣ 택배사: 송장번호 자동 반환
    ↓
6️⃣ 시스템: 주문에 자동 반영 + 셀러 알림
    ↓
7️⃣ 셀러: 송장 확인 → 자기 채널에 입력
```

### 4.3 AI 채팅 업무 처리 플로우

```
셀러: "주문 업로드할게" + 📎쿠팡_주문.xlsx
    ↓
AI: 엑셀 분석 → 컬럼 자동 인식 (고객명, 주소, 상품명, 수량 등)
    ↓
AI: "쿠팡 주문 15건 인식했습니다. 등록할까요?"
    ↓
셀러: "응"
    ↓
AI: 상품 매칭 + 가격 계산 + 주문 생성
    ↓
AI: "✅ 15건 등록 완료! 총 주문액 1,245,000원"
```

## 5. 기술 스택

### 5.1 프론트엔드
- **Framework**: Next.js 15 (App Router)
- **Language**: TypeScript
- **UI Library**: Tailwind CSS + shadcn/ui
- **State Management**: React Context / Zustand
- **Form Handling**: React Hook Form + Zod

### 5.2 백엔드
- **API**: Next.js API Routes
- **Database**: PostgreSQL
- **ORM**: Prisma
- **인증**: NextAuth.js (JWT 기반)
- **파일 처리**: xlsx 라이브러리

### 5.3 AI 및 실시간
- **AI**: Claude API (자연어 처리, 엑셀 자동 인식)
- **실시간 통신**: WebSocket (채팅)

### 5.4 외부 연동
- **택배 API**: CJ대한통운 API
- **알림**: 카카오 알림톡 API

### 5.5 인프라
- **웹 배포**: Vercel (→ 추후 AWS ECS 전환 용이)
- **WebSocket 서버**: Railway or Render (→ 추후 AWS EC2)
- **Database**: Supabase PostgreSQL (→ 추후 AWS RDS)
- **파일 스토리지**: Supabase Storage (→ 추후 AWS S3)

> 💡 **확장성 고려**: 초기에는 Vercel + Supabase로 빠른 개발 및 배포를 진행하고, 트래픽 증가 시 AWS로 손쉽게 전환 가능하도록 설계합니다.

## 6. 개발 일정 (6주)

## 🎯 프로젝트 배경 및 목적

### 현재 상황

대건로지온은 대형차 택배 배차를 전문으로 하는 물류 기업으로, 다음과 같은 업무 프로세스를 운영하고 계십니다:

- **다채널 견적 요청 접수**: 관계사, 프랜차이즈 업체 등으로부터 카카오톡을 통해 배차 견적서 요청 접수
- **다중 플랫폼 조회**: 전국24시콜화물, 화물맨, 원콜, 인성데이타 등 4개 화물 플랫폼을 개별적으로 확인
- **수작업 견적 비교**: 각 플랫폼의 견적을 수동으로 비교하여 최적 견적 선정
- **메뉴얼 기반 업무**: 요일별 부서별 업무 메뉴얼을 참조하여 업무 수행
- **수동 견적서 작성**: 고객사별로 견적서를 일일이 작성하여 제공
- **분산된 고객사 정보**: 엑셀, 수첩 등에 산재된 고객사 정보 관리

### 해결 과제

1. **사용자 및 권한 관리 부재**: 직원별 역할과 권한이 체계적으로 관리되지 않음
2. **고객사 정보 분산**: 고객사 정보가 여러 곳에 흩어져 있어 관리 어려움
3. **분산된 업무 환경**: 카카오톡, 4개 화물 플랫폼을 각각 확인해야 하는 비효율성
4. **수동 데이터 입력**: 동일한 배차 정보를 4개 사이트에 반복 입력하는 시간 소모
5. **메뉴얼 접근성**: 부서별 업무 매뉴얼을 찾고 확인하는 시간 지연
6. **견적 비교의 어려움**: 여러 플랫폼의 견적을 비교하여 최적안을 선택하는 복잡도
7. **견적서 작성 시간**: 고객사별 견적서를 수동으로 작성하는 반복 작업

### 프로젝트 목표

**통합 배차 관리 플랫폼 구축**을 통해:
- ✅ 체계적인 사용자 및 권한 관리로 보안 강화
- ✅ 통합 고객사 관리로 정보 일원화
- ✅ 카카오톡 견적 요청 파일을 자동으로 시스템에 업로드
- ✅ 4개 화물 플랫폼 API 통합으로 한 번의 입력으로 전체 플랫폼 조회
- ✅ AI 챗봇 기반 실시간 메뉴얼 제공으로 업무 효율성 향상
- ✅ **표준 견적서 자동 생성**으로 견적서 작성 시간 90% 단축
- ✅ 데이터 기반 의사결정 기반 마련

### 1. 사용자 및 고객사 관리 시스템 ⭐ 우선순위 1

- **계정 정보 관리**:
  - 프로필 정보 (이름, 부서, 직급, 연락처)
  - 프로필 이미지 업로드
  - 비밀번호 변경

- **관리자 기능**:
  - 직원 계정 생성/수정/삭제
  - 계정 일괄 등록 (CSV/Excel)
  - 계정 활성화/비활성화

  - 마지막 로그인 시간 추적

##### 1.1.2 로그인 보안

- 로그인 이력 기록

#### 1.2 권한 관리 시스템 (RBAC - Role-Based Access Control)

##### 1.2.1 역할(Role) 정의
- **슈퍼 관리자 (Super Admin)**:
  - 모든 기능 접근 가능
  - 사용자 계정 관리
  - 시스템 설정 관리
  - 권한 설정 관리

- **관리자 (Admin)**:
  - 대부분 기능 접근 가능
  - 고객사 관리
  - 견적서 발행/수정/삭제
  - 통계 조회
  - 직원 계정 조회 (생성/삭제 불가)

- **일반 사용자 (User)**:
  - 기본 업무 기능만 접근
  - 견적 조회
  - 견적서 발행 (승인 필요)
  - AI 챗봇 사용
  - 본인 프로필 수정

- **조회 전용 (Viewer)**:
  - 읽기 권한만 보유
  - 모든 데이터 조회 가능
  - 수정/삭제/생성 불가

##### 1.2.2 세부 권한 설정
- **메뉴별 접근 제어**:
  | 메뉴 | 슈퍼관리자 | 관리자 | 일반사용자 | 조회전용 |
  |------|-----------|--------|-----------|---------|
  | 대시보드 | ✅ | ✅ | ✅ | ✅ |
  | 파일 업로드 | ✅ | ✅ | ✅ | ❌ |
  | 4개 플랫폼 조회 | ✅ | ✅ | ✅ | ✅ |
  | 견적서 발행 | ✅ | ✅ | 승인필요 | ❌ |
  | 고객사 관리 | ✅ | ✅ | 조회만 | 조회만 |
  | AI 챗봇 | ✅ | ✅ | ✅ | ✅ |
  | 통계/리포트 | ✅ | ✅ | 부분 | ✅ |
  | 사용자 관리 | ✅ | 조회만 | ❌ | ❌ |
  | 시스템 설정 | ✅ | ❌ | ❌ | ❌ |

- **기능별 권한 (CRUD)**:
  - **Create (생성)**: 데이터 생성 권한
  - **Read (조회)**: 데이터 조회 권한
  - **Update (수정)**: 데이터 수정 권한
  - **Delete (삭제)**: 데이터 삭제 권한

- **데이터 범위 권한**:
  - 전체 데이터 접근 (관리자)
  - 부서별 데이터만 접근 (부서장)
  - 본인 생성 데이터만 접근 (일반 사용자)

##### 1.2.3 부서별 권한 관리
- 부서 생성/수정/삭제
- 부서별 메뉴 접근 권한 설정
- 부서별 데이터 접근 범위 설정
- 부서장 지정 및 관리

##### 1.3.1 고객사 기본 정보 관리

  - 태그별 통계

- **고객사 상태**:
  - 활성 (정상 거래)
  - 휴면 (3개월 이상 거래 없음)

##### 1.3.4 고객사 특이사항 관리
- **메모 및 특이사항**:
  - 고객사별 메모 (자유 작성)

- **파일 첨부**:
  - 계약서 스캔본

  - 기타 참고 문서

##### 1.3.5 고객사 검색 및 필터링
- **검색 기능**:
  - 고객사명 검색
  - 사업자등록번호 검색
  - 담당자 검색
  - 전화번호 검색
  - 통합 검색 (전체 필드)

- **필터링**:
  - 등급별 필터
  - 상태별 필터
  - 태그별 필터
  - 계약 유형별 필터
  - 등록일/최근 거래일 기간 필터

- **정렬**:
  - 고객사명 가나다순
  - 등록일 최신순
  - 거래 금액 높은순
  - 최근 거래일 순

##### 1.3.6 고객사 대시보드
- 총 고객사 수
- 등급별 고객사 분포
- 신규 고객사 (이번 달)

- **백엔드**: Node.js + Express / Python + FastAPI
- **파일 스토리지**: AWS S3 / NCP Object Storage
- **OCR**: Google Cloud Vision API / Naver Clova OCR (선택)
- **데이터베이스**: PostgreSQL

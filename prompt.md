# 상세페이지 제작 AI 드림팀 워크플로우 (콤팩트 버전)

> 3명의 전문 AI 에이전트가 **병렬 협업**하여 후킹 → 전환까지 완벽한 상세페이지를 제작합니다
> Claude Code의 **Team 기능**을 활용하여 에이전트 간 의존성 기반 병렬 처리를 수행합니다

---

## 0. 실행 방법

### 한 줄 실행

```
prompt.md 파일을 실행해줘
```

### 필요 입력: 상품 URL 또는 상품 정보

```
- 상품 URL: [URL] (자동 수집)
또는
- 상품명, 카테고리, 가격, 핵심 특징 등 직접 입력
```

## 0.5. 사전 브리핑 (에이전트 작업 전 필수)

> **상품 정보를 수집한 직후, 에이전트를 투입하기 전에** 아래 질문을 사용자에게 합니다.
> 각 질문마다 상품 특성에 맞는 **추천안 1~3개를 먼저 제시**하고, 사용자가 선택하거나 직접 입력합니다.

### 질문 항목 (AskUserQuestion 도구 사용)

#### Q1. 카피 톤 & 말투
상품 카테고리와 타겟을 분석하여 적합한 톤을 추천합니다.

**추천안 예시 (상품별로 달라짐):**
| 추천 | 톤 | 특징 | 적합한 상황 |
|------|-----|------|------------|
| A (추천) | 친근 구어체 | "~하셨죠?", "~거든요" | 식품, 생활용품, 대중 타겟 |
| B | 전문가 톤 | "~입니다", 데이터 중심 | 건강기능식품, 테크, 전문 타겟 |
| C | 감성 스토리텔링 | 서사 중심, 감정 호소 | 선물, 프리미엄, 감성 소비 |

> 팀 리드가 상품을 분석한 뒤 가장 적합한 톤을 (추천)으로 표시합니다.

#### Q2. 비주얼 방향
상품의 카테고리, 가격대, 경쟁 환경에 따라 추천합니다.

**추천안 예시 (상품별로 달라짐):**
| 추천 | 방향 | 특징 | 적합한 상황 |
|------|------|------|------------|
| A (추천) | 어두운 프리미엄 | 블랙/다크 배경, 고급 조명 | 고기, 주류, 프리미엄 식품 |
| B | 밝고 깨끗한 | 화이트/크림 배경, 자연광 | 건강식품, 유아용품, 뷰티 |
| C | 비비드 컬러풀 | 원색 강조, 팝한 느낌 | 간식, MZ 타겟, 트렌드 상품 |

#### Q3. 핵심 강조 포인트
어떤 USP를 가장 전면에 내세울지 결정합니다.

**추천안 예시 (상품별로 달라짐):**
| 추천 | 포인트 | 전략 |
|------|--------|------|
| A (추천) | 가격 메리트 | 할인율/가성비를 첫 화면에 강조 |
| B | 품질/원산지 | 프리미엄 품질을 시각적으로 증명 |
| C | 편의성/간편함 | "바로 사용 가능"의 즉시 만족 강조 |

#### Q4. 참고 레퍼런스 (선택)
```
비슷한 톤의 상세페이지나 브랜드가 있으면 URL 또는 설명을 알려주세요.
없으면 건너뛰어도 됩니다.
```

### 브리핑 결과 → 에이전트에 전달

사용자 답변을 정리하여 **모든 에이전트 프롬프트에 아래 블록을 추가**합니다:

```
## 사전 브리핑 (사용자 지정)

- 참고 레퍼런스: [있으면 URL/설명, 없으면 "없음"]

⚠️ 위 브리핑 내용을 반드시 반영하여 작업하세요.
```

## 1. 팀 아키텍처

### 에이전트 구성 (3명)

```
┌───────────────────────────────────────────┐
│            team-lead (팀 리드)              │
│     태스크 생성, 의존성 관리, 스토리보드 생성   │
├───────────────────────────────────────────┤
│                                             │
│  [1] strategist  → 전략 + 심리 통합 기획자    │
│  [2] copywriter  → 카피라이터      ← 병렬    │
│  [3] designer    → 비주얼 디자이너  ← 병렬    │
│                                             │
└───────────────────────────────────────────┘
```

### 태스크 의존성 & 병렬 처리 구조

```
[Task #1] 전략 + 심리 통합 브리핑 (strategist)
    │
    ├──────────────────┐
    ▼                  ▼
[Task #2] 카피 작성  [Task #3] 비주얼 기획     ← #1 완료 후 동시 병렬 실행
(copywriter)         (designer)
    │                  │
    └──────┬───────────┘
           ▼
[Task #4] 최종 통합 + 검수 (team-lead)         ← #2, #3 모두 완료 후
```

> 기존 5단계 → **3단계**로 압축. 대기 시간 대폭 단축.

---

## 2. 팀 생성 & 태스크 설정 절차

### STEP 1: 팀 생성

```
TeamCreate:
  team_name: "detail-page-team"
  description: "상세페이지 제작 AI 드림팀"
```

### STEP 2: 상품별 output 디렉토리 자동 생성

```bash
PROJECT_DIR="output/design/detail/$(date +%Y%m%d)_[상품명]"
mkdir -p "$PROJECT_DIR/.work"
```

**폴더 구조:**
```
output/design/detail/[YYYYMMDD]_[상품명]/
├── final_detail_page.md    ← 최종 기획서 (사용자용)
├── storyboard.html         ← 스토리보드 (브라우저에서 미리보기)
└── .work/                  ← 중간 파일 (기획 완료 후 자동 삭제)
    ├── 01_strategy_brief.md
    ├── 02_copy.md
    └── 03_visual_guide.md
```

### STEP 3: 태스크 4개 생성 + 의존성 설정

| Task ID | Subject | Owner | Blocked By | Output File |
|---------|---------|-------|------------|-------------|
| #1 | 전략 + 심리 통합 브리핑 | strategist | - | `{PROJECT_DIR}/.work/01_strategy_brief.md` |
| #2 | 6존별 후킹 카피 작성 | copywriter | #1 | `{PROJECT_DIR}/.work/02_copy.md` |
| #3 | 비주얼 기획 & 나노 바나나 프롬프트 | designer | #1 | `{PROJECT_DIR}/.work/03_visual_guide.md` |
| #4 | 최종 통합 기획서 + 검수 | team-lead | #2, #3 | `{PROJECT_DIR}/final_detail_page.md` |
| #5 | HTML 스토리보드 생성 | team-lead | #4 | `{PROJECT_DIR}/storyboard.html` |

### STEP 4: 에이전트 순차/병렬 투입

```
1단계: strategist 투입 → Task #1 완료 대기
2단계: copywriter + designer 동시 투입 (병렬) → Task #2, #3 모두 완료 대기
3단계: 팀 리드가 직접 Task #4, #5 수행 (통합 + 검수 + 스토리보드)
```

### STEP 5: 정리 & 팀 종료

```
1. 전체 에이전트에 shutdown_request 전송
2. .work/ 폴더 삭제 (중간 파일 보존 옵션 선택 시 유지)
3. TeamDelete로 팀 정리
4. 사용자에게 안내:
   - output/design/detail/[날짜]_[상품명]/final_detail_page.md (최종 기획서)
   - output/design/detail/[날짜]_[상품명]/storyboard.html (브라우저에서 스토리보드 미리보기)
```

## 4. 레퍼런스: 고객 구매 여정 & 6 ZONE 구조

> 에이전트들이 참고하는 핵심 프레임워크입니다.

### 고객 내면 대화 7단계

| 단계 | 시점 | 내면 대화 | 실패 시 |
|------|------|----------|---------|
| 1 시선 멈춤 | 0~3초 | "어? 이게 뭐지?" | 즉시 이탈 |
| 2 공감 | 3~10초 | "나도 이 문제 있어" | 스크롤 중단 |
| 3 관심 | 10~30초 | "이런 게 있구나" | "나중에 볼게" |
| 4 의심 | 30~60초 | "진짜 효과 있어?" | 경쟁사로 이탈 |
| 5 욕구 | 60초~ | "갖고 싶다" | 장바구니만 담음 |
| 6 망설임 | 구매 직전 | "지금 사야 하나?" | "다음에 사야지" |
| 7 안심 | 최종 | "실패해도 괜찮겠지?" | 결제 직전 포기 |

### 심리 전환 포인트 (각 단계 사이의 벽)

```
1→2: "나와 상관없잖아?" → 타겟의 구체적 상황/고민 언급
2→3: "뻔한 얘기" → 예상 깨는 솔루션 or 놀라운 데이터
3→4: "광고 아닌가?" → 제3자 증거 (리뷰, 인증, 판매 데이터)
4→5: "다른 게 더 낫지?" → 경쟁 대비 차별점 + 비교표
5→6: "지금 꼭 사야 해?" → 한정 수량, 기간 한정 할인
6→7: "잘못 사면?" → 무료 반품, 100% 환불 보장
```

### 6 ZONE 요약

|------|------|------|------|----------|
| 1 | 후킹 | 0~3초 | 스크롤 멈추기 | 후킹 헤드라인 + 히어로 이미지 |
| 2 | 문제 인식 | 3~10초 | "나도 그래!" | 페인포인트 + 손실 회피 |
| 3 | 솔루션 | 10~30초 | "괜찮은데?" | USP + 경쟁 비교 + 앵커링 |
| 4 | 신뢰 | 30~60초 | "믿을 만하네" | 사회적 증거 + 권위 |
| 5 | 전환 | 60초~ | "지금 사야겠다!" | CTA + 긴급성 + 희소성 |

## 5. 레퍼런스: 카피 공식 & 나노 바나나

### 카피 공식 5선

| 공식 | 구조 | 적합 상품 |
|------|------|----------|
| PAS | 문제 → 자극 → 해결 | 고민 해결형 |
| AIDA | 주의 → 관심 → 욕구 → 행동 | 신제품, 혁신 기능 |
| BAB | Before → After → Bridge | 라이프스타일 변화형 |
| 4U | 긴급 → 고유 → 구체적 → 유용 | 프로모션, 한정판 |
| FOMO | 놓치면 후회 강조 | 한정 수량, 시즌 |

### 나노 바나나 (Gemini 이미지 생성)

**모델 선택:**
| 구분 | Nano Banana | Nano Banana Pro |

| 용도 | 제품컷, 배경, 라이프스타일 | 텍스트 배너, 인포그래픽 |
| 속도 | 3~5초 | 10~20초 |
| 텍스트 | 약함 | 강함 (한글 OK) |

**프롬프트 작성 원칙:**
